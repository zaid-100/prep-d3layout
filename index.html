D3 Layout

Layouts in D3 are not visuals themselves but rather ways to change or map data in a new way and generate new data convenient for a visual task. There are 3 main types of Layouts discussed in chapter 13.

  1. Pie
  2. Stack
  3. Force

Pie

Pie changes the data so that a pie-chart can be visualized using D3

Stack

Stack changes the data so visualizations like Bar Charts can be made using D3.

Force

Used to change data so a "force" graph can be made using D3.

The commonality

Although each of these D3 Layouts have different syntax to present the data and have different purposes, all of the map functions themselves merely transform the data from one form to another so that D3 syntax can be used later to do its job. This is done when binding the data to our function which visualizes the data in our SVG. It is as follows: 

 svg.selectAll("g")
.data(d3.pie(dataset))
.enter()
.append("g")

Note that the layout function is just d3.pie() used in .data() phrase of the D3 algorithm. we will see similar method used to transform data in Stack and Force Layouts as well:

stack 
var groups = svg.selectAll("g")
.data(d3.stack(dataset))
.enter()
.append("g")
Force 
 var force = d3.forceSimulation(dataset.nodes)
.force("charge", d3.forceManyBody())
.force("link", d3.forceLink(dataset.edges))
.force("center", d3.forceCenter().x(w/2).y(h/2))

Although the data is only changed when binding, some setup variables are defined for force as shown above and then linked while binding data as shown below:

svg.selectAll("line")
.data(dataset.edges)
.enter()
.append("line")

Note that the D3 code for force when we defined that variable is running and the ".force" phrase which says d3.forceLink(dataset.edges) already links the data being bound to the visual element together in a force diagram, so that it appears as it is supposed to.
